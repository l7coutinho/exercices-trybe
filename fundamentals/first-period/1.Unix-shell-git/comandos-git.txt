Alguns comandos mais importantes git-github

git init - inicia o projeto
git clone - clona o projeto do diretorio remoto
git add - serve para adicionar pastas ao repositório
git commit - Armazena o conteúdo atual do índice em um novo commit, juntamente com uma mensagem de registro do usuário que descreve as mudanças. Se usa o commit depois de já ter feito o git add, para fazer o commit:
git status - A principal ferramenta utilizada para determinar quais arquivos estão em quais estados é o comando:
git rm - Para remover um arquivo do Git, você tem que removê-lo dos arquivos que estão sendo monitorados
git branch "name" -  Isso cria uma nova "pasta" para o mesmo commit em que você está no momento.
git checkout "name" - serve para alterar a pasta do commit
git merge "name" - serve para puxar atualizações feitas em outro branch.
git pull - serve para atualizar seu diretório a partir do diretorio remoto. Atualizando local
git push - é o comando em que você transfere commits a partir do seu repositório local para um repositório remoto.

Alguns comandos para navegar pelo terminal

cd (diretorio onde deseja entrar) - para acessar pastas do seu diretório local
ls - lista todas as pastas do diretorio atual
mkdir - cria um novo diretorio
touch - cria arquivos no diretorio
rm (nome do arquivo) - deleta o arquivo
rmdir - deleta o diretorio
mv - usado para mover arquivos ou diretorios e também para renomear.
cp - usado para copiar arquivos ou diretorios
